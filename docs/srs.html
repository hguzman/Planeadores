Especificación de requerimientos

1. Introducción

En este documento se provee información acerca del propósito, alcance, definiciones, acrónimos, abreviaturas, referencias y estructura del SRS

1.1 Proposito

El SRS describe completamente el comportamiento externo de la aplicación o sistema identificado. También describe requerimientos no funcionales, restricciones de diseño y factores necesarios que den una descripción comprensiva de los requerimientos para el software.

1.2 Alcance

Se creará un aplicativo web que ayudará a la empresa Planeadores a agilizar los procesos internos y tener un mejor control sobre los clientes y ventas por esto se hará necesario la creación de una base de datos.

2. Descripción general

2.1 Perspectiva del producto

El aplicativo web será un producto diseñado para trabajar en entornos WEB, lo que permitirá su utilización de forma descentralizada

2.2 Funcionalidad del producto

  1. Administración de Agentes: Los usuarios del sistema podrán gestionar los agentes (agregar, modificar, eliminar, buscar)
  2. Administración de planeaciones: Los usuarios del sistema podrán gestionar las planeaciones de los agentes (agregar, modificar, eliminar, buscar)
  3. Administración de transacciones: Los usuarios del sistema podrán gestionar las transacciones de los agentes (agregar, modificar, eliminar, buscar)
  4. Administración de Usuarios: El administrador podrá gestionar los usuarios que podrán acceder al sistema (agregar, modificar, eliminar, buscar)

2.3 Caracteristicas de los usuarios

El aplicativo web contará con dos tipos de usuarios administrador y usuario normal.
